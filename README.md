# Godot2Amiga

> Build classic Commodore Amiga games using Godot as the front-end and ACE as the runtime.

Godot2Amiga is an open-source export pipeline that converts a portable `.g2a`
game package into a native Amiga executable.

The long-term goal is to allow developers to prototype games in Godot while
targeting classic Amiga hardware (starting with OCS/ECS) through a deterministic,
fully open toolchain.

# Toolchain

```
Godot
   │
   ▼
.g2a package
   │
   ▼
validate
   │
   ▼
build
   │
   ▼
assets
   │
   ▼
compile
   │
   ▼
pack
   │
   ▼
run
```

Everything after the `.g2a` package is completely independent of Godot and can
be used by other engines or content pipelines.

# Quick Start

Clone the repository:

```bash
git clone https://github.com/Godot2Amiga/godot2amiga.git

cd godot2amiga
```

Install dependencies:

```bash
uv sync
```

Configure the toolchain:

```bash
g2stack doctor
```

Run the ACE showcase (recommended first test):

```bash
g2stack showcase --build
```

Build and run the example package:

```bash
g2stack dev examples/minimal.g2a
```

# Example Workflow

Validate a package:

```bash
g2a validate examples/minimal.g2a
```

Generate an ACE project:

```bash
g2a build examples/minimal.g2a
```

Convert assets:

```bash
g2a assets examples/minimal.g2a
```

Compile:

```bash
g2a compile build/minimal
```

Package:

```bash
g2a pack build/minimal
```

Run:

```bash
g2stack run build/minimal/dist
```

Or simply:

```bash
g2stack dev examples/minimal.g2a
```

# Philosophy

Godot2Amiga deliberately separates the authoring environment from the runtime.

```
Godot
      │
      ▼
   .g2a package
      │
      ▼
Godot2Amiga Compiler
      │
      ▼
ACE Runtime
      │
      ▼
Native Amiga executable
```

The `.g2a` package is intended to become a stable interchange format that can
eventually be generated by engines other than Godot.
